API Design Docs for Emedix Backend 

Users - 
POST /api/auth/get-otp -
1. get reqeust body - mobile number
2. Validate mobile number format.
3. Generate 6 digit OTP.
4. Hash OTP (never store plain OTP).
5. Store in DB/Redis with expiry (5 minutes).
6. Send OTP using SMS provider (Twilio / MSG91 / Fast2SMS etc).
7. Send response { success: true, message: "otp sent successfully" }

POST /api/auth/verify-otp - 
1. get request body - mobile number and otp
2. Fetch OTP record for mobile.
3. Check expiry.
4. Compare hashed OTP.
    If correct:
        Delete OTP record
        Create / fetch user
        Generate JWT access token
    If wrong:
        Increase attempt_count
5. Send response - Success.

POST /api/auth/resend-otp
get request body – mobile number
Fetch user record for mobile number.
Generate new 6 digit OTP.
Hash OTP.
Replace old otp_hash with new one.
Update expires_at (5 minutes from now).
Reset attempt_count = 0.
Send OTP using SMS provider (Twilio / MSG91 / Fast2SMS etc).
Send response
{
  "success": true,
  "message": "otp resent successfully"
}

POST /api/auth/logout
get request header – Authorization token
Validate JWT token.
If token valid → logout user on client side.
No database update required.

Send response:

{
  "success": true,
  "message": "logout successful"
}

GET /api/auth/me
get request header – Authorization token
Validate JWT token.
Extract mobile number or user id from token.
Fetch user from database.

Send response:

{
  "success": true,
  "data": {
    "mobile": "9876543210",
    "created_at": "2026-04-01T10:30:00Z"
  }
}
