API Design Docs for Emedix Backend 

Users - 
POST /api/auth/get-otp -
1. get reqeust body - mobile number
2. Validate mobile number format.
3. Generate 6 digit OTP.
4. Hash OTP (never store plain OTP).
5. Store in DB/Redis with expiry (5 minutes).
6. Send OTP using SMS provider (Twilio / MSG91 / Fast2SMS etc).
7. Send response { success: true, message: "otp sent successfully" }

POST /api/auth/verify-otp - 
1. get request body - mobile number and otp
2. Fetch OTP record for mobile.
3. Check expiry.
4. Compare hashed OTP.
    If correct:
        Delete OTP record
        Create / fetch user
        Generate JWT access token
    If wrong:
        Increase attempt_count
5. Send response - Success {
                   "success": true,
                   "message": "OTP verified",
                   "data": {
                     "accessToken": "jwt_token_here",
                     "user": {
                       "id": "u_123",
                       "mobile": "9876543210"
                     }
                   }
                 }
                 - Fail {
                          "success": false,
                          "message": "Invalid or expired OTP"
                        }
