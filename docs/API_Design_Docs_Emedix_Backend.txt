API Design Docs for Emedix Backend (Current System)

Endpoints - 

POST /api/auth/get-otp
1. Get request body: mobile_no
2. Validate mobile number (10-digit).
3. Generate 6-digit OTP.
4. Hash OTP with bcrypt.
5. Store in Redis with 300-second (5 min) expiry.
    - Key: "otp:{mobile_no}" | Value: Hashed OTP.
6. Send OTP using Twilio API.
7. Send Response: { "success": true, "message": "otp sent successfully" }

POST /api/auth/verify-otp
1. Get request body: mobile_no and otp.
2. Fetch hashed OTP from Redis for the given mobile_no.
3. Check if record exists (if not, OTP expired or not requested).
4. Compare submitted OTP with hashed OTP from Redis.
    If Correct:
        Delete OTP from Redis.
        Check MySQL for User with mobile_no.
        If User doesn't exist: Create new User in MySQL (NO GHOST USERS before this step).
        Generate JWT Access Token.
    If Incorrect:
        Throw Unauthorized error (Invalid OTP).
5. Send Response: { "success": true, "data": { "accessToken": "...", "user": { "id": "...", "mobile_no": "..." } } }

POST /api/auth/resend-otp
1. Get request body: mobile_no.
2. Re-trigger "get-otp" logic.
3. Redis automatically overwrites the old OTP with a fresh expiry.
4. Send new OTP via Twilio.
5. Send Response: { "success": true, "message": "otp sent successfully" }

GET /api/auth/me
1. Get request header: Bearer [JWT_TOKEN].
2. Validate JWT token using JwtAuthGuard.
3. Extract user ID (sub) from token payload.
4. Fetch User profile from MySQL database.
5. Send Response:
{
  "success": true,
  "data": {
    "id": "...",
    "mobile_no": "9876543210",
    "created_at": "..."
  }
}

1️⃣ Get User Current Location
POST /api/address/get-location
Request Body
{
  "latitude": 17.385044,
  "longitude": 78.486671
}
Steps
Get request body:
latitude
longitude
Validate:
latitude range (-90 to 90)
longitude range (-180 to 180)
Call Google Maps Geocoding API (Reverse Geocoding)
lat,lng → actual address
Extract structured fields:
formatted_address
city
state
country
pincode
Insert into addresses table

Fields stored:

user_id
formatted_address
city
state
country
pincode
latitude
longitude
source = "gps"
Send response
{
  "success": true,
  "data": {
    "formatted_address": "MG Road, Hyderabad, Telangana, India",
    "city": "Hyderabad",
    "state": "Telangana",
    "pincode": "500001",
    "latitude": 17.385044,
    "longitude": 78.486671
  }
}
Production Errors

Possible failures:

Invalid lat/long
Google API quota exceeded
Google API network timeout
Location in ocean / invalid region

Return:

{
  "success": false,
  "error": "Unable to resolve address from location"
}
2️⃣ Add Address Manually
POST /api/address/add
Request Body
{
  "address_line": "12 MG Road",
  "city": "Hyderabad",
  "state": "Telangana",
  "pincode": "500001"
}
Steps
Validate input fields

Required:

address_line
city
state
pincode
Build full address string
"12 MG Road Hyderabad Telangana 500001"
Call Google Maps Geocoding API (Forward Geocoding)
address → lat,lng
Validate response

Reject if:

ZERO_RESULTS
Extract:
latitude
longitude
formatted_address
Insert into addresses table
source = "manual"
Response
{
  "success": true,
  "data": {
    "formatted_address": "12 MG Road, Hyderabad, Telangana 500001",
    "latitude": 17.38,
    "longitude": 78.48
  }
}
Production Errors

Possible cases:

INVALID_ADDRESS
ADDRESS_NOT_FOUND
GEOCODING_TIMEOUT
